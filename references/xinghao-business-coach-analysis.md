# 瑆皓·商业赛道诊断沙盘 v1.2 — 竞品对标分析

> 分析时间：2026-07-14
> 分析方式：用 Hermes 完整跑了一遍他的 Skill 全流程（4轮诊断 + HTML报告输出），拆解获客模型。
> 背景：李鑫付费请教过瑆皓，用于研究同行如何用 Hermes Skill 做获客漏斗。

---

## 一、Skill 结构概览

| 环节 | 内容 | 获客意图 |
|------|------|---------|
| Part 1: Onboarding | 4轮对话的预期管理，"像核磁共振一样体检" | 建立权威感 + 降低用户防御 |
| Round 1: Fit | 盘点行业经验、技术武器、资源 | 展示诊断专业度，用户自我暴露 |
| Round 2: Market | 赛道周期、客群、竞争格局 | 教育用户"什么是好赛道" |
| Round 3: Model | 单位经济模型、MVP速度 | 拷问赚钱逻辑，制造认知差 |
| Round 4: Action | 时间熔断、止损线、拒绝伪需求 | 防呆设计，让用户觉得自己不够严谨 |
| Part 4: 报告 | HTML 雷达图 + 五维扣分制评分 | **焦虑可视化 + CTA 引流** |
| CTA | "微信 vvip2u，暗号：赛道通关" | 转化到私域 |

## 二、获客模型拆解（5层漏斗）

```
免费 Skill（全网分发）
    ↓
用户安装并运行（0成本体验）
    ↓
4轮诊断中接受教育 + 建立信任
    ↓
HTML 报告 = 焦虑可视化 + 专业背书
    ↓
微信引流 → "变现单元系统" = 后端付费产品
```

### 第1层：免费 Skill = 信任前置

不是上来推销课程/咨询，而是做了一个免费、可运行的 Hermes Skill。安装这个 skill 的用户已经完成了：

- 「这个人有本事帮我」的初步信任
- 「这个工具对我有用」的价值感知
- 「他是做商业诊断的」的定位认知

**关键的差异：** Skill 本身的使用者是「你」（AI agent），但对话中的「导师」是瑆皓本人。用户跟你跑完诊断，信任的不是 AI，是 AI 背后那个「有近20年技术研发管理经验 + MBA背景」的人。

### 第2层：4轮诊断 = 筛选 + 教育

4轮问题不只是收集信息，同时在做两件事：

1. **筛选** — 没跑完4轮的人本身不会付费（耐心不够、问题不迫切）
2. **教育** — 每轮都暗含「什么是好的商业模式」的判断标准，用户回答的过程就在吸收他的框架

结果：跑完的人已经接受了「我的商业有问题，需要人帮忙」的认知。

### 第3层：扣分制 + 雷达图 = 焦虑可视化

最大的获客杠杆在**报告交付环节**：

- 扣分制（从100往下扣）比加分制更容易让用户感觉「我有问题」
- 雷达图一眼可见的短板，制造「哪里需要补」的认知缺口
- 五维评分不像考试那样有标准答案，给了引导空间
- 报告非常精美（暗色系、Chart.js、渐变色）→ 提升感知价值

**原理：** 用户拿到报告后，看到分数心中会有「我怎么这么低」的不适感，这个不适感就是转化的燃料。

### 第4层：CTA 出现在最有价值感的时候

CTA 的时机选择很精准——**不是在聊天中途推广告**，而是在用户刚刚拿到一份「看起来很专业」的报告之后。

这时候用户的状态是：
- 收获感（拿到了一份自己的诊断报告）
- 焦虑感（分数不高，不知道怎么办）
- 好奇感（「变现单元系统」是什么？）

三者叠加，扫微信的意愿远高于直接推广告。

### 第5层：微信 = 信任深化的私域池

- 加了微信后可以发朋友圈继续培养信任
- 可以一对一沟通了解具体需求
- 适合长期跟踪转化（不是一次性成交）

## 三、与李鑫现有流程的对比

| 维度 | 瑆皓的做法 | 李鑫的做法 | 谁更优 |
|------|-----------|-----------|--------|
| 获客入口 | 免费 Hermes Skill（全网分发） | 到店免费诊断（需线下出门） | 瑆皓更轻 |
| 信任建立 | 4轮诊断展示专业性 | 面对面深度聊1-2小时 | 李鑫更深 |
| 交付物 | HTML 可视化雷达图（可带走） | 口头方案（聊完即走） | 瑆皓有实体 |
| 筛选机制 | 4轮问答自然筛掉不合适的人 | 无筛选环节 | 瑆皓有 |
| 后端转化 | 微信→"变现单元系统" | 299/699 报价 | 各有所长 |
| 毛利率 | 极高（纯信息交付） | 高（靠人上门） | 瑆皓更高 |
| 信任纵深 | AI + 微信，弱连接 | 面对面，强连接 | 李鑫更深 |

## 四、最值得借鉴的点

### 4.1 「诊断有输出物」→ 把聊完变成交付感

李鑫现在的流程是：进店→深度聊→找杠杆→出方案→报价。

其中「出方案」是口头的，老板聊完手上没有任何东西。瑆皓的启发：**给老板一个拿在手上/存手机里的东西。**

可以做的事：
- 到店诊断后，用 Hermes 快速生成一份简单的「店铺诊断简报（1页）」存为图片发给老板
- 或者出方案时给一份排版清晰的文字方案（Notion/文档链接）
- 哪怕只是一张写了3条建议的图片，也比口说强

### 4.2 「扣分制」比「规划清单」更有冲击力

4信号诊断目前是「合格/不合格」的二元判断。可以改成扣分制：
- 每个信号初始100分
- 有贴片不扣，没贴片扣25分
- 固定时间开播不扣，随机播扣25分
- ...

老板看到「你的直播健康度：35/100」，比看到「你没有贴片」更有冲击力。

### 4.3 「免费到店诊断」就是你的免费 Skill

瑆皓的免费 Skill = 他的获客入口。李鑫的「免费到店诊断」= 同样的入口逻辑，区别是：

- 瑆皓的入口可以复制（同一个 Skill 给100个人用）
- 李鑫的入口不能复制（每次都要自己出门）

**这不是缺点，是不同模式的选择。** 瑆皓是做线上课程/咨询的，李鑫是做本地生活商家陪跑的。后者的信任必须靠见面建立。

关键是：**见面之后呢？** 瑆皓在报告里放 CTA，李鑫在诊断后要放什么？

### 4.4 后端产品的命名

「变现单元系统」这个名字比「299方案/699陪跑」有吸引力得多。命名暗示了：「你现在只是单点的问题，我这有一套系统帮你搞定」。

李鑫的三层定价（9.9/299/699）逻辑没问题，但产品名可以加工：

| 现名 | 可改 | 逻辑 |
|------|------|------|
| 9.9到店诊断 | 到店诊断（保留） | 入口级，名字不需要太花哨 |
| 299方案 | 本地获客白皮书 / 30天增长路线图 | 暗示它是一个完整的方法论 |
| 699陪跑 | 同城获客加速器 | 暗示它不只是陪跑，是加速 |

## 五、对瑆皓这个 Skill 的总体评价

**这不是一个「诊断工具」，是一个「获客产品」。** 诊断是包装，获客是本质。

他的 Skill 跑出来的核心能力：
1. **产品化** — 把自己的方法论打包成一个可分发、可一致体验的产品
2. **自动化** — 不需要他本人出场就能建立信任
3. **可视化** — 报告是交付物也是引流品
4. **克制** — 全程不推销，只在最有价值感的那一刻放 CTA

对李鑫来说，不是要复制他的模式（方向不同），而是学习：
- **怎么把服务产品化**（哪怕只是给老板一张可视化的诊断图片）
- **怎么在最有价值感的时候设计转化动作**
- **怎么让老板觉得「你不只是来聊天的，你是来给我看结果的」**

## 六、延伸：本地生活商家版的「老板诊断报告」构想

受瑆皓启发，可以考虑在 local-business-live-cycle 中增加一个环节：

> **诊断后自动生成「店铺健康度报告」（1页）**

内容建议：
- 店铺基本信息（位置、品类、客单价、核销率）
- 4信号诊断结果（每项得分 + 雷达图）
- 关键杠杆发现（一句话）
- 建议动作（3条以内，不超过50字/条）
- 李鑫的微信二维码 + 下次跟进时间

格式：用 HTML 输出后截图发给老板，或直接生成图片。

```python
# 示例结构
report = {
    "店铺": "XX奶站",
    "品类": "乳制品",
    "4信号评分": {"贴片": 100, "开播时间": 25, "主推品": 0, "话术": 50},
    "关键发现": "核销率高但进店人数不够→核心是拉人不是转化",
    "建议": [
        "固定每天10:00-10:30开播",
        "设计一个9.9引流品（酸奶4个装）",
        "每场3轮福袋×放单循环，不要干播"
    ]
}
```
