# 本地生活商家直播全流程

> 你说一句话，AI 帮你做。不需要看书，不需要学方法。

### 🏃 快速入口

| 你想做什么 | 直接说 | AI 会做什么 |
|-----------|--------|-----------|
| 🏪 诊断门店 | 「帮我诊断门店」 | 问20个问题，出诊断报告（评分+短板+3条建议） |
| 📝 写口播视频 | 「写视频」 | 选Hook→写稿→5铁律检查→出完整脚本 |
| 📡 准备开播 | 「帮我准备一场直播」 | 根据你的店+引流品，出完整直播框架（节奏+话术+福袋+选品） |
| 🎤 播前话术 | 「帮我写直播话术」 | 按3轮节奏生成逐字话术 |
| 📊 复盘数据 | 「复盘昨天的直播」/「复盘那条视频」 | 分析数据→找问题→改进行动 |
| 🎯 学具体技巧 | 「福袋怎么发」/「什么时候放单」 | 直接回答，不绕弯 |
| 🆘 卡住了 | 「不会用」 | 引导你回正路 |

> 不知道先做什么？说「帮我诊断门店」，从诊断开始。

## 🚀 自助诊断模式（店主直接使用）

> 你是店主。你不需要懂直播术语，不需要懂运营。回答下面的问题，AI 会自动帮你生成一份《门店诊断报告》，告诉你问题在哪、该怎么做。

### 如何开始

直接说 **「帮我诊断门店」** 或粘贴以下问题的答案。按顺序回答完所有问题后，AI 会：

1. 给你一个 **直播健康度评分**（0-100）
2. 指出你的 **最短板**（只打一个点，不打十个）
3. 生成 **3条可执行建议**（明天的第一件事）

---

### 问题清单（共 6 组，约 20 题）

#### 📋 A 组：门店基本面（3题）

| # | 问题 | 选项 |
|---|------|------|
| A1 | 你是什么类型的店？ | 餐饮 / 美业 / 酒旅 / 亲子 / 零售 / 其他 |
| A2 | 现在每天大概有多少客人进店？ | 0-10人 / 10-30人 / 30-50人 / 50人以上 |
| A3 | 平均客单价大概多少？ | 0-20元 / 20-50元 / 50-100元 / 100元以上 |

#### 📋 B 组：线上意识（3题）

| # | 问题 | 选项 |
|---|------|------|
| B1 | 你开过抖音/美团直播吗？ | 没开过 / 试过几次 / 定期在播 |
| B2 | 你有发短视频吗？ | 没发过 / 偶尔发 / 定期在发 |
| B3 | 你有微信私域群吗？ | 没有 / 有但没怎么管 / 有且维护着 |

#### 📋 C 组：引流品（3题）

| # | 问题 | 选项 |
|---|------|------|
| C1 | 你现在有团购/外卖链接在线上卖吗？ | 有 / 没有 |
| C2 | 如果有，最便宜的引流品卖多少钱？ | 1-9.9元 / 10-29.9元 / 30元以上 / 没有 |
| C3 | 顾客到店一般还会再买点别的吗？ | 基本只核销团购 / 偶尔会加 / 来了就不空手走 |

#### 📋 D 组：直播质量（4题）— 如果你在播

| # | 问题 | 选项 |
|---|------|------|
| D1 | 直播时有贴片（店名+福利+地址）吗？ | 有 / 没有 / 没播不知道 |
| D2 | 你固定时间开播吗？ | 固定每天 / 固定偶尔 / 想播就播 / 没播 |
| D3 | 直播时有主推的品吗？ | 有明确的 / 随口讲 / 没有 / 没播 |
| D4 | 直播话术有框架吗？ | 有框架 / 大概知道 / 想到哪说到哪 / 没播 |

#### 📋 E 组：到店体验（3题）

| # | 问题 | 选项 |
|---|------|------|
| E1 | 顾客到店后核销流程方便吗？ | 很方便 / 还可以 / 有点麻烦 / 不清楚 |
| E2 | 店里有地方让顾客多待一会儿吗？ | 有座位/体验区 / 只有柜台 / 纯外卖 |
| E3 | 店里有东西值得顾客拍照发朋友圈吗？ | 有 / 一般 / 没有 |

#### 📋 F 组：精力意愿（2题）

| # | 问题 | 选项 |
|---|------|------|
| F1 | 你每天能花多少时间在线上？ | 1小时以上 / 30分钟 / 基本没时间 |
| F2 | 你觉得自己店未来的最大问题是？ | 没人知道 / 人来了留不住 / 利润太薄 / 竞争太大 |

### 诊断逻辑（AI 自动执行，不用店主看）

收到以上答案后，按以下流程出报告：

**Step 1: 直播健康度评分**
- 只评 D 组（有直播数据时）。公式同上（4信号扣分制）。
- 没播过的店，跳过评分，直接到 Step 2。

**Step 2: 到店承接能力判断**
- E1（核销方便）+ E2（体验空间）+ E3（拍照价值）综合判断
- 如果 ≥ 6分（满分9分） → **店能接住人，问题在拉人**
- 如果 < 6分 → **店接不住人，先解决店的问题再拉人**

**Step 3: 找关键杠杆（只打一个点，不列清单）**

| 条件 | 杠杆 |
|------|------|
| 店能接人 + 没播过 | 先开播，用福袋+引流品拉人到店 |
| 店能接人 + 在播但效果差 | 优化直播节奏（见第二章） |
| 店接不住人 | 先改核销流程/体验（不改的话拉来人也留不住） |
| 没时间打理线上 | 锁定1个最小的动作（如：每天拍1条手机视频） |

**Step 4: 出 3 条行动建议**
- 仅 3 条。按顺序执行，第一条是明天早上能做的。
- 格式：
  ```
  你的门店诊断结果
  ─────────────────
  直播健康度：XX分（如果你在播）
  最大短板：一句话说清楚
  店能不能接人：能 / 不能

  建议你明天做这三件事：
  1. [最小的动作，明天就能做]
  2. [本周内完成]
  3. [本周内完成]

  ─────────────────
  有疑问？去反馈页面提交
  https://app.notion.com/p/3a28278657fa81a796e8fc39dd5d5055
  ─────────────────

  问题想清楚了，比方法更重要。
  ```

## 第一章：线上获客（短视频引流）

> 没有短视频就没有新观众。直播间自然流太慢，出圈短视频是唯一增量来源。

### 1.1 532内容配比

> 📌 **框架参考**（来源：陈晶IP方法论）

| 类型 | 占比 | 目的 | 拍什么（本地生活版） |
|------|------|------|-------------------|
| 出圈类 | 5条 | 让没看过你的人看见你 | 老板反差/行业秘密/同城热点 |
| 干货类 | 3条 | 让看见你的人觉得你专业 | 怎么选店/怎么避坑/成本揭秘 |
| 人设类 | 2条 | 让觉得你专业的人信任你 | 帮过的案例/老板故事 |

### 1.2 发布节奏

```
开播前2小时：发1条出圈类（为直播蓄水）
开播时：     发1条干货类（预告今日主题）
直播中：     每小时发1条切片（直播精彩片段）
```

### 1.3 "说人话"短视频文案5条铁律

> 📌 **框架参考**（来源：Vikki内容方法论）

| # | 铁律 | 操作方法 |
|---|------|----------|
| 1 | **脑海有画面** | 写完朗读，自己脑海里能形成具体画面吗？说"好吃"没感觉，说"金灿灿的鸡皮、一股烟往上飘"才有感觉 |
| 2 | **三年级听得懂** | 找目标用户里理解力最弱的人，在心里问他：他能听懂吗？ |
| 3 | **有具体细节** | 不能只写"成本高"，要写"少了一块板、工人再跑一趟、客户三天后再请假回来" |
| 4 | **有故事** | 人类大脑靠共同经验理解抽象概念，故事才能激活想象 |
| 5 | **两个圆重叠** | 你说的 ≠ 他听到的。写完问自己：用户真的收到这个信息了吗？ |

### 1.4 出圈类选题模板

| 钩子类型 | 适合本地生活的示例 |
|---------|------------------|
| 名人反差 | "XXX火锅老板说赚钱靠服务，但你知道他的真实成本是多少吗？" |
| 反认知 | "开了几家店反而越赚越少，你信不信？" |
| 借势热点 | "XX品牌一夜关了XX家店，本地商家能学到什么？" |

### 1.5 付费引流补充

- **谁需要用**：自然流稳定后，需要破层级/修正标签
- **理想占比**：控制在30%以内，理想约20%

## 第二章：开播获客（福袋×放单循环）

> 本章基于商家实战方案。适用场景：商家有实体店、有团购品、核销率尚可，但直播间没人进。

### 2.1 整场节奏（30-40分钟）

适合本地生活获客型直播——不长篇大论，每轮10分钟，3轮搞定：

```
第1轮 0-10min  发福袋→等人进→塑品→放引流品单
第2轮 10-20min 开福袋→发新福袋→逼单→推升单换购
第3轮 20-30min 开福袋→发新福袋→最后冲刺→收尾
```

### 2.2 核心逻辑

```
福袋不是奖品，是买停留时长的工具。
        ↓
福袋在抽的时候，人就有理由留在直播间等开奖。
        ↓
人齐了才放单——福袋没开之前不放单。
        ↓
放单后立刻预告下一轮福袋——让人继续等。
```

**关键话术：** "说不定中奖的就是你，中了就不用下单了"——这句让观众有理由留下。

### 2.3 逐轮话术框架

**第1轮（0-10min）：蓄水+首单**

| 时间 | 动作 | 话术要点 |
|------|------|---------|
| 0:00-0:30 | 开场钩子 | "XX的邻居们，我是楼下XX店老板。今天不整虚的，上来先送福利" |
| 0:30-1:00 | 发福袋 | "左上角福袋发了XX福利，点关注亮灯牌就能参与，倒计时5分钟" |
| 1:00-3:00 | 等福袋期间做停留 | "福袋在抽着，说不定中奖的就是你，中了就不用下单了" |
| 3:00-4:00 | 塑品 | "平时去超市买多少钱？今天我补贴——XX价格" |
| 5:00-6:00 | 放单 | "同城福利，拍了去店里拿。链接在下面，点进去直接拍" |
| 6:00-8:00 | 升单钩子 | "到店核销时加X元换购XX——这个是到店才有的福利" |
| 8:00-10:00 | 预告下一轮 | "没拍到的别急，下一轮福袋马上跟上" |

**第2轮（10-20min）：续流+逼单**

- 发新福袋
- 等福袋期间重复塑品+逼单
- 强调到店核销引导
- 福袋开了之后追单

**第3轮（20-30min）：冲刺+收尾**

- 最后一轮福袋
- "还剩最后X份"
- 总结：拍了去哪领、中奖了去哪领
- 预告下次直播时间

### 2.4 开播前准备清单

```
□ 手机架好，背景露出店招
□ 引流品链接已上架
□ 福袋已设置（奖品+5分钟倒计时）
□ 私域群已发预告
□ 开播前2小时发一条预告短视频
```

### 2.5 直播中红线

```
❌ 不要开场就卖货——先发福袋等人进
❌ 福袋没开之前不要放单——放了单人就走了
❌ 每轮放单后不预告下一轮——让人没有理由继续等
❌ 读数据、看在线人数分心——专注说话，数据下播看
```

### 2.6 关键数据目标

| 指标 | 目标 |
|------|------|
| 引流品核销率 | ≥ 60% |
| 到店升单率 | ≥ 30%（老板自己干） |
| 单场新增关注 | ≥ 50个 |
| 平均停留时长 | ≥ 60秒 |

## 第三章：复盘

> 复盘有两个入口，根据你的场景选择：

> **入口一：「复盘直播」** — 你有完整直播数据（场观、转化、GPM等），走六部曲深度分析（数据→节奏→商品→人群→短视频→流量结构）
>
> **入口二：「复盘视频」** — 你发了短视频想知道效果，走内容迭代闭环（盲预测→发布→T+3d复盘→升级标准）
>
> 不确定走哪个？说「复盘」，AI自动判断。

### 3.1 复盘六部曲（适合直播数据完整时）

> 📌 **框架参考**（来源：那村儿风起短视频内容）

```
数据复盘 → 节奏复盘 → 商品复盘 → 人群复盘 → 短视频复盘 → 流量结构复盘
   ↓          ↓          ↓          ↓          ↓            ↓
定基调     定成败     找爆款     判精准     挖流量       调比例
```

**第一步 数据复盘：** 看转化率、GPM/客单价、老粉成交占比、停留时长。标出不合格指标。

**第二步 节奏复盘：** 开播极速流接住了吗？放单量是否超过在线5%？是否有连续2-3轮成交递增？

**第三步 商品复盘：** 商品曝光点击率15%合格25%+爆款。曝光高点击低→话术/弹窗问题；点击高转化低→价格/信任问题。

**第四步 人群复盘：** 对比看播vs购买人群画像。无主力人群→需微付费修正标签。

**第五步 短视频复盘：** 筛选基础播放>5000、点击率>5%、5秒完播>30%的视频→开播前抖加加热。

**第六步 流量结构复盘：** 对比各渠道流量占比和支付金额占比。微付费自然流50-70%健康。

**六步的递进逻辑：**
- Step 1-2 → 诊断"直播间本身"的问题（数据基线和节奏）
- Step 3-4 → 诊断"货和人"的问题（商品效率和人群精度）
- Step 5-6 → 诊断"流量来源"的问题（短视频引流和整体结构）

> **📎 拓展参考**
> - `external/marketing-livestream-commerce-coach.md` — 直播电商主播教练（数据复盘/直播间调优/主播训练）
> - `external/marketing-douyin-strategist.md` — 抖音策略师（流量运营/算法思维/付费流机制）
>
> 所有外部 skill 见 `external/` 目录。

### 3.2 本地生活商家关键指标速查

| 指标 | 合格线 | 说明 |
|------|--------|------|
| 曝光进入率 | ~20% | 大众类目~20%；小众10-20% |
| 商品曝光率 | 80-90% | 中控弹窗效率 |
| 商品点击率 | 15%+ | 20%有潜力，25%+爆款 |

## 第五章：营销活动设计

> 本章提供营销活动的设计框架与参考，涵盖引流品设计、大促策划、裂变获客等场景。内容引用自行业通用方法论，尚未经李鑫完整实战验证。

### 5.1 引流品设计

> 📌 **框架参考**（来源：`sub-skills/sales-offer-lead-gen-strategist` — Offer 与 Lead Gen 策略师）

引流品不是"便宜货"，是一个完整的offer设计。offer的价值由四个变量决定：

```
              梦想结果 × 达成的感知可能性
价值 = ────────────────────────────────────
                时间延迟 × 付出与牺牲
```

**对本地生活商家的应用：**

| 变量 | 本地生活版操作 | 示例（奶站门店） |
|------|--------------|----------------|
| 梦想结果 | 用买家自己的语言描绘结果 | "来店里喝到当天新鲜的酸奶" |
| 感知可能性 | 用保证和证据降低怀疑 | "不好喝全额退" |
| 时间延迟 | 压缩购买到使用的时间 | "今天下单，今天就能来店里拿" |
| 付出与牺牲 | 移除用户的决策门槛 | 不用注册，不用关注，到店报手机号即可 |

**引流品设计检查清单：**
- [ ] 顾客看到价格时，会觉得"这点钱不买亏了"吗？
- [ ] 引流品和到店的升单品有明确关联吗？
- [ ] 引流品的核销流程是否足够简单？（不要三个步骤）
- [ ] 有没有风险逆转？（不好吃退、不满意换）

### 5.2 大促策划参考

> 📌 **框架参考**（来源：`sub-skills/marketing-china-ecommerce-operator` — 中国电商运营专家）

本地生活商家的大促不是618/双11，是**会员日/店庆/换季清仓/新品尝鲜**。电商大促的玩法可以降维使用：

| 电商玩法 | 本地生活版 | 适用场景 |
|---------|-----------|---------|
| 预售定金膨胀 | 预存返现 | 高客单服务（美容套餐、私教课） |
| 跨店满减 | 多买多折 | 标品类（酸奶买5送1，买10送3） |
| 限时秒杀 | 每天一个爆品限量 | 短效品（当天到期酸奶） |
| 拼团 | 老带新优惠 | 拉新获客 |
| 优惠券 | 到店核销券 | 所有品类 |

**执行步骤：**

```
第1步 定目标：这场活动要拉新？清库存？提利润？
第2步 选玩法：根据产品类型选上面的玩法
第3步 算账：折扣后还有利润吗？引流品亏多少能接受？
第4步 定节奏：蓄水期几天？爆发期？收尾期？
第5步 拍素材：提前拍好活动预告短视频
第6步 开播执行：按直播节奏走
第7步 复盘：核销率、新客占比、利润核算
```

### 5.3 裂变获客参考

> 📌 **框架参考**（来源：`sub-skills/marketing-growth-hacker` — 增长黑客）

本地生活商家最适合的裂变机制：

| 机制 | 原理 | 示例 |
|------|------|------|
| **邀请奖励** | 老客带新客，双方得利 | "带朋友来店，每人减5元" |
| **分享解锁** | 分享后获得额外福利 | "发朋友圈，到店送一份小食" |
| **拼团** | 多人购买更便宜 | "3人拼团，每人19.9（原价29.9）" |

**裂变的核心原则：**
- 分享动作要自然不尴尬（不要求朋友注册/下载）
- 奖励要即时兑现（到店核销时直接给）
- 裂变门槛要低（一个分享动作换一个福利）

## 第六章：口播视频文案 + 内容迭代

> 诊断做完，知道店的问题了。现在用短视频把人拉进来。这一章帮你写出能拉人的口播视频稿，还能越写越好。

### 6.1 写口播视频稿

#### Step 1：确定本期选题

| 你的目标 | 对应的选题方向 |
|----------|---------------|
| 让更多人知道我的店 | 出圈类：店的故事/老板反差/行业秘密 |
| 让人觉得我专业 | 干货类：怎么选店/怎么避坑/成本揭秘 |
| 直接拉人到店 | 促销类：限时福利/新品体验/到店送 |
| 随便，你帮我想 | 根据诊断结果推荐 |

**与诊断结果联动：** 如果报告说「店能接人，问题在拉人」→ 选促销类。如果「店接不住人」→ 先改店，再拍视频。

#### Step 2：选Hook公式（开头5种）

| 公式 | 适合场景 | 示例（餐饮版） |
|------|---------|-------------|
| 🏷️ **标签型** — 地域+品类锁定周边 | 日常更新，快速获客 | 「很多杭州的朋友刷到我，刻意来我家吃火锅」 |
| 🔢 **数字型** — 具体数字制造信息差 | 干货/揭秘 | 「月流水30万，到手不到3万，钱去哪了？」 |
| ⚡ **冲突型** — 认知反差 | 出圈爆款 | 「千万别买XX，除非你看完这条」 |
| 🎁 **利益型** — 直接给好处 | 促销活动 | 「花9.9吃了3个菜，你敢信？」 |
| 💗 **共鸣型** — 戳痛点 | 建立信任 | 「每次请客都找不到合适的店？我也是」 |

**如何选：** 促销活动 → 利益型／标签型。出圈涨粉 → 冲突型／数字型。日常维护 → 标签型／共鸣型。

#### Step 3：用Vikki 5条铁律写正文

写完开头后，用下面5条检查每一句。**不过就改到过为止。**

| # | 铁律 | 错误写法 → 正确写法 |
|---|------|-------------------|
| 1 | **脑海有画面** | ❌「好吃」→ ✅「金灿灿的鸡皮、一股烟往上飘」 |
| 2 | **三年级听得懂** | ❌「严格品控」→ ✅「每天早上6点我去市场亲手挑的」 |
| 3 | **有具体细节** | ❌「服务好」→ ✅「上次有客人感冒，老板娘熬了碗姜汤端过去」 |
| 4 | **有故事** | ❌「食材新鲜」→ ✅「客人说这鱼就像刚从河里蹦上来的」 |
| 5 | **两个圆重叠** | ❌「全场5折」→ ✅「到店才有、限今天、每人限1份」 |

**写完自动执行5铁律检查。** 每发现一条不通过的，自动改写后问你是否满意。

#### Step 4：输出格式

```markdown
## 标题：[用户会点进来的标题]

### 开头（前3秒）
[Hook公式类型：标签型]
「[开头第一句]」

### 正文（30-60秒）
[按Vikki铁律检查过的人话内容]
- 先说具体场景（让脑海有画面）
- 再说你的店/产品（用故事串起来）
- 最后说为什么跟观众有关（两个圆重叠）

### 结尾（3秒）
- 引导动作：「[位置]，[福利]，等你来」
- 评论互动：「你遇到过吗？评论区聊聊」

### 5铁律自检 ✅✅✅✅✅
```

### 6.2 内容迭代闭环

> 发完不是结束。**真正值钱的是下次能不能写得更好。**

#### 6.2.1 盲预测（发布前）

发布前做一次盲预测，**写完不能改。** 目的：训练你对「什么内容能火」的判断力。

| 维度 | 问自己 | 打分(1-10) |
|------|--------|-----------|
| 开头吸引力 | 前3秒会让人停下来吗？ | |
| 人话程度 | 三年级能听懂吗？ | |
| 情绪张力 | 看完有感觉吗（好奇/共鸣/惊讶）？ | |
| 到店驱动 | 看完想去你的店吗？ | |
| 原创性 | 别家店说过这个吗？ | |

**同时记录一条预测：** 「我猜这条播放会达到[多少]」

**关键规则：** 预测写进文件，发布后不能改。

#### 6.2.2 登记发布

```

```

#### 6.2.3 T+3d 复盘

3天后：
```
复盘那条水饺视频
```

| 对比项 | 预测值 | 实际值 | 偏差分析 |
|--------|--------|--------|---------|
| 播放量 | XX | XX | 猜高/猜低了？ |

| 场景 | 一线处理 | 还是不行 |
|------|---------|---------|
| 用户说「不知道拍什么」 | 问：「你店里最近最忙的时候是什么时候？有没有什么特别的客人？」 | 推荐选题库中最通用的3个：「老板的一天」「店里最贵的XX」「客人最常问的问题」 |
| 用户说「写得太长了」 | 立即缩短到30秒，只留Hook+1个卖点+结尾 | 改为手机录制的"随手拍"模式，不追求完播率 |
| 用户说「我不卖这个价」 | 「价格你自己改，我给你框架：XXX店的招牌菜，原价XX，今天XX」 | 不写价格，只写画面描述和卖点 |
| 用户说「太难了不会拍」 | 「手机打开录像，对着店里最热闹的地方拍15秒，不用剪辑」 | 改为文字海报形式，截图发朋友圈就行 |
| 盲预测不知道打多少分 | 「保守点打6分，然后发出去看看，3天后复盘」 | 默认给中间值5分，先跑起来 |

### 🛠️ 自助诊断异常处理（fallback）

| 场景 | 一线处理 | 还是不行 |
|------|---------|---------|
| 用户说「太多了不想回答」 | 「没关系，先回答3个核心问题就行：你是什么店、一天大概多少人、开过直播吗」 | 直接给通用建议：「建议先发短视频，每天1条手机拍的店里的日常」 |
| 用户回答模糊（「还行吧」「一般」） | 追问具体数字：「没事，大概数就行，10个人还是50个人？」 | 用行业平均值代替，在报告末尾标注「部分数据按行业均值估算」 |
| 用户跳过所有问题 | 直接问：「那我先给你一个通用的起步建议：每天拍1条视频，内容就拍店里最热闹的瞬间」 | 引导去 Notion 反馈页 |
| 报告太长看不懂 | 「我帮你浓缩成一句话：你现在最大的问题是[XX]，明天早上先做[XX]」 | |
| 用户说「这些我都知道」 | 「那我直接出建议：你的店能接人，现在只差拉人进来。要不要我帮你写第一条视频稿？」 | |

| 不要做 | 为什么 |
|--------|--------|
| 一次写太长（超过90秒） | 本地生活视频30-60秒完播率最高 |
| 开头不说人话 | 前3秒抓不住人，后面白写 |
| 只写产品不写场景 | 顾客在乎产品能帮他解决什么，不是产品本身 |
| 发完就不管了 | 不复盘，下一期跟上一期没区别 |
| 发之前改预测 | 预测不准没事，改预测才是问题 |
