## 🚀 自助诊断模式（店主直接使用）

> 你是店主。你不需要懂直播术语，不需要懂运营。回答下面的问题，AI 会自动帮你生成一份《门店诊断报告》，告诉你问题在哪、该怎么做。

### 如何开始

直接说 **「帮我诊断门店」** 或粘贴以下问题的答案。按顺序回答完所有问题后，AI 会：

1. 给你一个 **直播健康度评分**（0-100）
2. 指出你的 **最短板**（只打一个点，不打十个）
3. 生成 **3条可执行建议**（明天的第一件事）

### 诊断逻辑（AI 自动执行，不用店主看）

收到以上答案后，按以下流程出报告：

**Step 1: 直播健康度评分**
- 只评 D 组（有直播数据时）。公式同上（4信号扣分制）。
- 没播过的店，跳过评分，直接到 Step 2。

**Step 2: 到店承接能力判断**
- E1（核销方便）+ E2（体验空间）+ E3（拍照价值）综合判断
- 如果 ≥ 6分（满分9分） → **店能接住人，问题在拉人**
- 如果 < 6分 → **店接不住人，先解决店的问题再拉人**

**Step 3: 找关键杠杆（只打一个点，不列清单）**

| 条件 | 杠杆 |
|------|------|
| 店能接人 + 没播过 | 先开播，用福袋+引流品拉人到店 |
| 店能接人 + 在播但效果差 | 优化直播节奏（见第二章） |
| 店接不住人 | 先改核销流程/体验（不改的话拉来人也留不住） |
| 没时间打理线上 | 锁定1个最小的动作（如：每天拍1条手机视频） |

**Step 4: 出 3 条行动建议**
- 仅 3 条。按顺序执行，第一条是明天早上能做的。
- 格式：
  ```
  你的门店诊断结果
  ─────────────────
  直播健康度：XX分（如果你在播）
  最大短板：一句话说清楚
  店能不能接人：能 / 不能

  建议你明天做这三件事：
  1. [最小的动作，明天就能做]
  2. [本周内完成]
  3. [本周内完成]

  ─────────────────
  有疑问？去反馈页面提交
  https://app.notion.com/p/3a28278657fa81a796e8fc39dd5d5055
  ─────────────────

  问题想清楚了，比方法更重要。
  ```

### 0.1 第一步：标准化进店调研（结构化访谈）

> 替代"随便问、随口说"。详见 `references/diagnosis-questions.md` 完整问题清单。

进店后按 **6 个模块** 自然推进，每个模块对应一个判断目标：

| 模块 | 目标 | 核心产出 |
|------|------|---------|
| **A: 基本面** | 判断经营热度（日均进店、客单价、复购率、趋势） | 店的基本面数据 |
| **B: 线上意识** | 判断老板的认知层次和执行力（试过没、信不信、愿意投入多少） | 转化难度等级 |
| **C: 产品与引流品** | 找到可用的引流品和到店升单路径 | 直播选品方向 |
| **D: 4信号诊断** | 如果已开播，评估直播质量（贴片/开播时间/主推品/话术） | 直播水平评估 |
| **E: 核销到店体验** | 判断"店能不能接住人"——方向分水岭 | 方案方向判断 |
| **F: 收尾与报价** | 总结发现 + 给方向 + 出报价 | 转化成交 |

**核心心法：**
- 不是审问，是对话。每个问题后给一句反馈（"哦，那说明你这个不错" / "这个确实是个问题"）
- 不要在模块 A 到 E 之间报价 — 还没建立价值认知就报价 = 等着被拒绝
- 如果老板话多，不用赶节奏
- 最终判断出方案方向后，落地详见：第二章（如果能接人，只做拉人）或 第六章（如果不能，先解决店的问题）

> 📎 完整问题清单见 `references/diagnosis-questions.md`

### 0.2 第二步：4信号扣分制诊断

> v1.3.0 升级：从"合格/不合格"二元判断改为扣分制。每个信号100分，老板看到的不是「你不行」，而是「你得了35分」——冲击力差5倍。

不看数据，看画面和操作。每个信号独立打分：

| 信号 | 扣分条件 | 不扣分条件 | 说明 |
|------|---------|-----------|------|
| 贴片 | 没有贴片（-100分） | 有贴片（店名+福利+地址+开播时间） | 贴片是直播间的招牌，没有=没人知道你在卖什么 |
| 开播时间 | 想播就播，没规律（-100分） | 固定时间开播（-0分），有固定习惯但偶尔断播（-50分） | 时间是信任，用户知道什么时候能刷到你 |
| 主推品 | 随口讲，没有主推品（-100分） | 有明确的引流品和主推品（-0分），有品但没讲清楚（-50分） | 没有主推品=没有爆款逻辑 |
| 话术 | 想到哪说到哪，没有框架（-100分） | 有框架有节奏（-0分），有框架但不够熟练（-50分） | 话术框架需要练习，但必须有 |

**计算方法：** `直播健康度 = 100 - 总扣分 / 4`

例：没贴片（-100）+ 固定时间（-0）+ 有主推（-0）+ 话术生疏（-50）= 扣150分 / 4 = 37.5分

**等级判定：**
- ≥80分：可以开播了，细节优化就行
- 50-79分：核心问题在流程，不是内容
- <50分：没把直播当正经事做，需要架构性的改变

> 💡 **从瑆皓学到的：** 扣分制比合格/不合格更有冲击力。老板看到"你的直播健康度：37/100"比看到"你没有贴片"刺痛得多。**先让他疼，他才会想治。**

### 0.3 第三步：找关键杠杆

老板娘说了一句关键信息：**"核销率高，来了都或多或少买点。"**

这句话决定了整个方案方向：

```
核销率高 + 到店有消费 → 店本身没问题
                 ↓
问题不是"转化"，是"进店人数不够"
                 ↓
直播间只干一件事：拉人进店
                 ↓
不需要复杂话术，不需要人设打造
只需要：福袋留人 + 低价引流品 + 到店核销引导
```

**这是你的判断模型：**
> 先看"店能不能接住人"——如果能，问题就是拉人，不是转化。
> 如果不能，才想话术、人设、信任的问题。

### 0.4 第四步：设计方案（实战版）

**引流品：** 9.9四个酸奶
- 前端不赚钱，目的是让人下单到店
- 成本结构：让老板自己算，你只设计流量逻辑

**直播节奏：** 3轮福袋×放单循环（30-40分钟）

| 轮次 | 动作 | 目的 |
|------|------|------|
| 第1轮 0-10min | 发福袋（10个鸡蛋）→等人进→塑品→放9.9酸奶单 | 蓄水+首单转化 |
| 第2轮 10-20min | 开福袋→发新福袋→逼单→推升单换购 | 续流+制造紧迫 |
| 第3轮 20-30min | 开福袋→发新福袋→最后冲刺→收尾 | 收割尾流 |

**核心逻辑：**
- 福袋是用来留人的工具，人齐了才放单
- 放单后立刻预告下一轮福袋——让人有理由继续等
- 福袋在抽的时候说"说不定中奖的就是你，中了就不用下单了"——这句话让观众有理由留在直播间等开奖

**升单：** 直播间不负责升单，只负责拉人进店。人到了店里，老板自己维护。

### 本地生活商家的特殊之处

| 区别 | 传统带货 | 本地生活商家 |

### 2026年抖音算法新规（影响本地生活商家）

```
推荐得分 = 价值观分 × 内容力权重 × 交易力权重 × 合规系数
```

- **内容力第一权重是场景**：实景溯源（门店实拍/工厂/产地）可获30%流量加权
- **推流机制从实时反馈转向7天慢推流**
- 第1天冷启动：不核心考核GMV，核心考核3秒留存、停留(>25秒)、互动率(>2%)、转粉率(>1.5%)
- 第4-5天：考核复访率、铁粉互动、GPM（需达客单价10倍以上）
- 核心数据权重：3秒留存 > 停留时长 > 老粉互动(权重比新粉高6倍) > 评论互动 > UV价值&GPM > 转化率

### 1.2 发布节奏

```
开播前2小时：发1条出圈类（为直播蓄水）
开播时：     发1条干货类（预告今日主题）
直播中：     每小时发1条切片（直播精彩片段）
```

### 1.3 "说人话"短视频文案5条铁律

> 📌 **框架参考**（来源：Vikki内容方法论。李鑫尚未实战验证每一条）

| # | 铁律 | 操作方法 |
|---|------|----------|
| 1 | **脑海有画面** | 写完朗读，自己脑海里能形成具体画面吗？说"好吃"没感觉，说"金灿灿的鸡皮、一股烟往上飘"才有感觉 |
| 2 | **三年级听得懂** | 找目标用户里理解力最弱的人，在心里问他：他能听懂吗？ |
| 3 | **有具体细节** | 不能只写"成本高"，要写"少了一块板、工人再跑一趟、客户三天后再请假回来" |
| 4 | **有故事** | 人类大脑靠共同经验理解抽象概念，故事才能激活想象 |
| 5 | **两个圆重叠** | 你说的 ≠ 他听到的。写完问自己：用户真的收到这个信息了吗？ |

### 1.5 付费引流补充

- **谁需要用**：自然流稳定后，需要破层级/修正标签
- **理想占比**：控制在30%以内，理想约20%
- **核心认知**：付费是放大流量的工具，成败根基在于直播间人货场和素材质量
- **本地生活注意**：同城定向要精准，避免浪费在非目标区域

## 第二章：开播获客（福袋×放单循环）

> 本章基于商家实战方案。适用场景：商家有实体店、有团购品、核销率尚可，但直播间没人进。
>
> ⚠️ 本节不包含连麦诊断/连线筛客内容——那些适用于周大侠合伙型直播间，不适用于本地生活获客型直播。

### 2.1 整场节奏（30-40分钟）

适合本地生活获客型直播——不长篇大论，每轮10分钟，3轮搞定：

```
第1轮 0-10min  发福袋→等人进→塑品→放引流品单
第2轮 10-20min 开福袋→发新福袋→逼单→推升单换购
第3轮 20-30min 开福袋→发新福袋→最后冲刺→收尾
```

### 2.2 核心逻辑

```
福袋不是奖品，是买停留时长的工具。
        ↓
福袋在抽的时候，人就有理由留在直播间等开奖。
        ↓
人齐了才放单——福袋没开之前不放单。
        ↓
放单后立刻预告下一轮福袋——让人继续等。
```

**关键话术：** "说不定中奖的就是你，中了就不用下单了"——这句让观众有理由留下。

### 2.3 逐轮话术框架

**第1轮（0-10min）：蓄水+首单**

| 时间 | 动作 | 话术要点 |
|------|------|---------|
| 0:00-0:30 | 开场钩子 | "XX的邻居们，我是楼下XX店老板。今天不整虚的，上来先送福利" |
| 0:30-1:00 | 发福袋 | "左上角福袋发了XX福利，点关注亮灯牌就能参与，倒计时5分钟" |
| 1:00-3:00 | 等福袋期间做停留 | "福袋在抽着，说不定中奖的就是你，中了就不用下单了" |
| 3:00-4:00 | 塑品 | "平时去超市买多少钱？今天我补贴——XX价格" |
| 5:00-6:00 | 放单 | "同城福利，拍了去店里拿。链接在下面，点进去直接拍" |
| 6:00-8:00 | 升单钩子 | "到店核销时加X元换购XX——这个是到店才有的福利" |
| 8:00-10:00 | 预告下一轮 | "没拍到的别急，下一轮福袋马上跟上" |

**第2轮（10-20min）：续流+逼单**

- 发新福袋
- 等福袋期间重复塑品+逼单
- 强调到店核销引导
- 福袋开了之后追单

**第3轮（20-30min）：冲刺+收尾**

- 最后一轮福袋
- "还剩最后X份"
- 总结：拍了去哪领、中奖了去哪领
- 预告下次直播时间

### 2.4 开播前准备清单

```
□ 手机架好，背景露出店招
□ 引流品链接已上架
□ 福袋已设置（奖品+5分钟倒计时）
□ 私域群已发预告
□ 开播前2小时发一条预告短视频
```

### 2.5 直播中红线

```
❌ 不要开场就卖货——先发福袋等人进
❌ 福袋没开之前不要放单——放了单人就走了
❌ 每轮放单后不预告下一轮——让人没有理由继续等
❌ 读数据、看在线人数分心——专注说话，数据下播看
```

### 2.6 关键数据目标

| 指标 | 目标 |
|------|------|
| 引流品核销率 | ≥ 60% |
| 到店升单率 | ≥ 30%（老板自己干） |
| 单场新增关注 | ≥ 50个 |
| 平均停留时长 | ≥ 60秒 |

### 📎 实战案例

完整逐字稿见 `CASE-某奶站门店.md`，含每句话的精确话术和操作清单。

## 第三章：复盘

### 3.1 复盘六部曲

> 📌 **框架参考**（来源：那村儿风起短视频内容。李鑫用过但未完整验证每一步的数字标准）

```
数据复盘 → 节奏复盘 → 商品复盘 → 人群复盘 → 短视频复盘 → 流量结构复盘
   ↓          ↓          ↓          ↓          ↓            ↓
定基调     定成败     找爆款     判精准     挖流量       调比例
```

**第一步 数据复盘：** 看转化率、GPM/客单价、老粉成交占比、停留时长。标出不合格指标。

**第二步 节奏复盘：** 开播极速流接住了吗？放单量是否超过在线5%？是否有连续2-3轮成交递增？

**第三步 商品复盘：** 商品曝光点击率15%合格25%+爆款。曝光高点击低→话术/弹窗问题；点击高转化低→价格/信任问题。

**第四步 人群复盘：** 对比看播vs购买人群画像。无主力人群→需微付费修正标签。

**第五步 短视频复盘：** 筛选基础播放>5000、点击率>5%、5秒完播>30%的视频→开播前抖加加热。

**第六步 流量结构复盘：** 对比各渠道流量占比和支付金额占比。微付费自然流50-70%健康。

**六步的递进逻辑：**
- Step 1-2 → 诊断"直播间本身"的问题（数据基线和节奏）
- Step 3-4 → 诊断"货和人"的问题（商品效率和人群精度）
- Step 5-6 → 诊断"流量来源"的问题（短视频引流和整体结构）

## 第五章：营销活动设计

> 本章提供营销活动的设计框架与参考，涵盖引流品设计、大促策划、裂变获客等场景。内容引用自行业通用方法论，尚未经李鑫完整实战验证。

### 5.1 引流品设计

> 📌 **框架参考**（来源：`sub-skills/sales-offer-lead-gen-strategist` — Offer 与 Lead Gen 策略师）

引流品不是"便宜货"，是一个完整的offer设计。offer的价值由四个变量决定：

```
              梦想结果 × 达成的感知可能性
价值 = ────────────────────────────────────
                时间延迟 × 付出与牺牲
```

**对本地生活商家的应用：**

| 变量 | 本地生活版操作 | 示例（奶站门店） |
|------|--------------|----------------|
| 梦想结果 | 用买家自己的语言描绘结果 | "来店里喝到当天新鲜的酸奶" |
| 感知可能性 | 用保证和证据降低怀疑 | "不好喝全额退" |
| 时间延迟 | 压缩购买到使用的时间 | "今天下单，今天就能来店里拿" |
| 付出与牺牲 | 移除用户的决策门槛 | 不用注册，不用关注，到店报手机号即可 |

**引流品设计检查清单：**
- [ ] 顾客看到价格时，会觉得"这点钱不买亏了"吗？
- [ ] 引流品和到店的升单品有明确关联吗？
- [ ] 引流品的核销流程是否足够简单？（不要三个步骤）
- [ ] 有没有风险逆转？（不好吃退、不满意换）

### 5.2 大促策划参考

> 📌 **框架参考**（来源：`sub-skills/marketing-china-ecommerce-operator` — 中国电商运营专家）

本地生活商家的大促不是618/双11，是**会员日/店庆/换季清仓/新品尝鲜**。电商大促的玩法可以降维使用：

| 电商玩法 | 本地生活版 | 适用场景 |
|---------|-----------|---------|
| 预售定金膨胀 | 预存返现 | 高客单服务（美容套餐、私教课） |
| 跨店满减 | 多买多折 | 标品类（酸奶买5送1，买10送3） |
| 限时秒杀 | 每天一个爆品限量 | 短效品（当天到期酸奶） |
| 拼团 | 老带新优惠 | 拉新获客 |
| 优惠券 | 到店核销券 | 所有品类 |

**执行步骤：**

```
第1步 定目标：这场活动要拉新？清库存？提利润？
第2步 选玩法：根据产品类型选上面的玩法
第3步 算账：折扣后还有利润吗？引流品亏多少能接受？
第4步 定节奏：蓄水期几天？爆发期？收尾期？
第5步 拍素材：提前拍好活动预告短视频
第6步 开播执行：按直播节奏走
第7步 复盘：核销率、新客占比、利润核算
```

### 5.3 裂变获客参考

> 📌 **框架参考**（来源：`sub-skills/marketing-growth-hacker` — 增长黑客）

本地生活商家最适合的裂变机制：

| 机制 | 原理 | 示例 |
|------|------|------|
| **邀请奖励** | 老客带新客，双方得利 | "带朋友来店，每人减5元" |
| **分享解锁** | 分享后获得额外福利 | "发朋友圈，到店送一份小食" |
| **拼团** | 多人购买更便宜 | "3人拼团，每人19.9（原价29.9）" |

**裂变的核心原则：**
- 分享动作要自然不尴尬（不要求朋友注册/下载）
- 奖励要即时兑现（到店核销时直接给）
- 裂变门槛要低（一个分享动作换一个福利）

### 5.5 常见陷阱

> 以下来自真实场景的教训，按优先级排列。

**陷阱1：直播没人时不要指望直播引流**
当本地生活商家直播每场只有十几单时，问题不是话术/排品/福袋——是根本没人看到。这时候把精力花在直播间就是浪费。正确的顺序：先短视频在同城炸一波 → 让更多人知道这个店 → 直播间只负责"活动通知站"和承接。

**陷阱2：不要给商家设计合规风险的动作**
如"每天限量30份"等话术在部分平台可能违规。出方案前先确认：这个话术/玩法在抖音本地生活规则里是否允许？不确定时走保守路线。

**陷阱3：引流品不要破坏正常价格体系**
低价清仓看起来能解燃眉之急，但抖音会记住商品的历史最低价。用换购/满赠/到店核销加赠的方式，比直降更安全，后续还能按正常价卖。

**陷阱6：报告数据纰漏——分数错了=信任崩塌**

> 🔴 **真实案例：鸿福醉仙鸭（2026-07-14）**
>
> 李鑫口头描述"鸭子19.9元利润10%"，我据此误判可以做9.9元引流品。被李鑫当场纠正：实际利润30%，9.9元完全不成立。报告重做两版。
>
> **教训：** 数据是报告的生命线。利润率、客单价、引流品成本——错一个，方案方向全偏。老板一眼看出"你连我赚多少钱都没搞清楚"——信任一秒崩塌。拿不准的数字先不说，跟李鑫确认再写入报告。

**陷阱7：不要替老板"发明"想法**

> 🔴 **真实案例：鸿福醉仙鸭（2026-07-14）**
>
> 我在报告里写"建议加炸串丰富产品线"。实际上加炸串是老板自己的想法。李鑫纠正：她已经在想了，不是你教她的。
>
> **教训：** 老板自己的好主意不要包装成你的建议去邀功。报告里要说"你自己也意识到了XX问题，这个方向是对的"——肯定老板的认知，同时展示你认同她。抢功是最快失去信任的方式。

**陷阱8：报告必须有分数环形图（不得省略）**

> 🔴 **真实案例：锦润鲜水饺报告（2026-07-14）**
>
> 我出锦润鲜水饺报告时做了简化版，去掉了顶部的分数环形图。李鑫马上指出"缺少了第一眼的分数和阶段"。分数环形图+雷达图是报告最核心的冲击力，省略了它报告就不完整了。
>
> **教训：** 报告格式必须**始终包含**：①分数环形图（top最优先）②雷达图 ③五维卡片 ④诊断结论 ⑤建议方案。环形图是老板看到的第一眼，也是情绪触发的起点——"28/100！这么低？"——这句话之后的对话才有成交的可能。

**陷阱4：诊断偏差——二手信息失真**

> 🔴 **真实翻车案例：天润乳业（2026-07-14）**
>
> 李鑫口头描述：店里直播从3单做到15单、老板愿意干、能到店承接——听起来是77分的"成熟门店"。
>
> 实际会议记录显示：老板有负债、部分门店员工半年多没发工资、正在上班、费用降了四五万后"播不播都不急"——真实只有64分，核心问题从"缺技术"变成了"不痛了"。
>
> **教训：** 当你从记忆中调取一个商家情况时，容易自动美化——你记得的是最好的那次状态。出报告前必须翻原始会议记录/通话录音，不要凭印象打分。报告分数错了 = 报价方向错了 = 信任崩塌。

**如何避免：**
1. 告诉 Hermes 商家情况时，附上原始会议记录文件路径，不要自己总结
2. 用"3-2-1"验证法：3个关键数据（日均进店、客单价、核销率）+ 2个老板的原话（他说了什么）+ 1个你的判断（问题到底是什么）
3. 如果老板"费用降了/不急了/不痛了" → 支付意愿信号 ⚠️ 红色预警

## 第六章：出诊报告输出（诊断完给老板看的）

> 诊断完不是聊完就走。给老板一份看得见的东西，他会觉得"这人真花了功夫"。
> 模板见 `templates/diagnosis-report-template.html`

### 6.1 什么时候出报告

**主流程：当场出（进店聊完 → 当面点开报告 → 指着报价）**

> 进店调研完（A-F模块走完），打开电脑/手机 Web UI，当场让 Hermes 生成报告。
> 老板看着报告上的雷达图和评分，新鲜热乎的时候直接报价。

**备选：回家出，第二天带去**

> 如果出门没带电脑，或者店里不方便（太吵/没网），就回家出，第二天再去。
> 带报告再去也有效果，但少了趁热打铁的情绪优势。

### 6.2 报告结构规范（必含6要素）

每份出诊报告必须包含以下6个部分，**缺一不可**：

```
┌─ 1. 门店信息 ─────────────────────────────┐
│   店名、诊断日期、诊断人                     │
├─ 2. 综合评分环形图 ────────────────────────┤
│   分数 + 阶段判定（萌芽/起步/成长/成熟/明星）  │
├─ 3. 五维雷达图 ────────────────────────────┤
│   线上意识 / 产品能力 / 直播执行 / 到店承接 / 执行意愿  │
├─ 4. 诊断结论 ─────────────────────────────┤
│   一段话说清楚"这家店现在是什么情况"          │
├─ 5. 五维评分卡片 ──────────────────────────┤
│   每个维度：分数 + 颜色标识 + 进度条 + 当前状态描述  │
└─ 6. 建议行动方案 ──────────────────────────┘
    ≤5条，每条可执行
```

**生成报告时用 HTML 模板 `templates/diagnosis-report-template.html`，按以下步骤：**
1. 根据调研数据给5个维度分别打分（1-100）
2. 计算总分 `round(五维平均分)`
3. 阶段判定：≤30萌芽 / 31-50起步 / 51-70成长 / 71-85成熟 / >85明星
4. 填写诊断结论（一段话，客观、正面、有方向）
5. 填写建议方案（≤5条，每条是具体动作）
6. 用 `write_file` 保存到桌面 `门店诊断报告-{店名}.html`
7. 用 Chrome 无头模式转 PDF `门店诊断报告-{店名}.pdf`
   ⚠️ Chart.js 必须关闭动画（`animation:{duration:0}`），否则 PDF 抓取时雷达图只渲染到一半。PDF命令加 `--virtual-time-budget=5000` 保证渲染完成。

### 6.3 5 维评分标准

诊断完用以下维度打分（每项 1-100），作为报告数据：

| 维度 | 高分信号（70+） | 低分信号（<50） |
|------|---------------|----------------|
| **线上意识** | 试过抖音/直播，愿意每天花1小时 | 完全没试过，觉得线上没用 |
| **产品能力** | 有明确引流品+升单品，价格合理 | 产品多而杂，没有爆品意识 |
| **直播执行** | 有贴片、定时开播、有主推品 | 干播、没规律、随口讲 |
| **到店承接** | 核销率高、到店会升单、有留存 | 核销率低、不升单 |
| **执行意愿** | 愿意投时间、愿意按方案做 | 只看不做、等着别人帮他干 |

### 6.4 报告生成步骤

```mermaid
1. 诊断完当场（或回家后）打开 Hermes
2. 告诉 Hermes：去XX店做了诊断，按调研情况描述
3. Hermes 自动：
   a. 按 6.3 评分标准打分
   b. 用 write_file 保存到桌面：~/Desktop/门店诊断报告-{店名}.html
   c. 输出文件路径给你
4. 当场点开桌面报告，指着给老板看
5. 报价
```

**关键原则：**
- 报告是"助攻"，不是"主角"——让报告帮你报价，不要让报告替你成交
- 报告展示时，指着每个维度跟老板解释——这个过程本身就是价值展示
- 报告里不要出现"后续收费XX元"这种话，报价只在口头说
- 报告发出后，如果老板连续 3 天没回应，直接跟进

### 6.5 实战心法：报告+报价话术

当场指着报告说（或者第二天带去说）：

> "老板，刚才聊的我帮你整理了一下，你看这个报告……（指着雷达图说）"
>
> "你店最大的问题在这——线上意识这块几乎空白，但到店承接其实挺好的，所以只需要做一件事：拉人进店。"
>
> "我帮你出一份详细的执行方案，你照着做就行。299 块钱，就一顿饭钱。我把我奶站那个方案的做法完全复制过来。"
>
> "或者你觉得方案自己懒得看，我上门陪你做两次，699，包含到店诊断+方案+两次上门陪拍陪播。"

> **核心：** 报告是你的"专业证明"，报价是"价值交换"。先让老板看见价值（报告），再报价——顺序不能反。

### 6.6 产品命名建议（提升感知价值）

> 📌 **来源：** 瑆皓诊断沙盘分析（`references/xinghao-business-coach-analysis.md`）——他的「变现单元系统」命名暗示了「你只是单点问题，我这有一套系统搞定」。

你的三层定价逻辑没问题，但产品名可以包装得更像"一套系统"而不是「单项服务」：

| 现在 | 可改 | 逻辑 |
|------|------|------|
| 9.9 到店诊断 | 到店诊断（保持） | 入口级，名字不需要花哨 |
| 299 方案 | **30天获客路线图** / **本地增长白皮书** | 暗示它是一个完整方法论，不是一张纸 |
| 699 陪跑 | **同城获客加速器** / **门店陪跑计划** | 暗示不只是陪跑，是加速增长 |

**原则：** 不包装成"你给我的服务"，包装成"你给我的系统/方案/计划"。老板付钱买的是"解决感"，不是"服务时数"。

### 6.7 核心原理：焦虑可视化

> 为什么这份报告能帮你成交？因为它把老板的"模糊焦虑"变成了"具体数字"。

**诊断前：** 老板觉得"生意不太好，也不知道差在哪"（模糊焦虑 → 不急着解决）

**看到报告后：** "线上意识 32/100？这么低？"（具体数字 → 痛点被量化 → 想治）

这个原理来自瑆皓的扣分制设计：老板看到分数的不适感就是转化的燃料。你在本地生活场景里做的是同样的事——把老板"说不清楚"的问题用数字炸出来。

你的报告要回答老板心里没说出来的三个问题：
1. **"我到底差在哪？"** → 雷达图 + 各维度得分
2. **"这个差对我有多大影响？"** → 诊断结论（"你店能接住人但没人来→你亏的是每天进店的钱"）
3. **"谁帮我解决？多少钱？"** → 你 + 报价

> 记住：老板付钱买的是"知道问题在哪并且有人能解决"的确定性——不是你的时间。

### 6.9 双品交叉销售策略（通用模式）

**核心逻辑：**

```

        ↓

        ↓

        ↓

```

**模式拆解：**

- 一个店做两个品类（堂食+零售、主食+副食）

## 第六章：口播视频文案 + 内容迭代

### 6.1 写口播视频稿

#### Step 1：确定本期选题

| 你的目标 | 对应的选题方向 |
|----------|---------------|
| 让更多人知道我的店 | 出圈类：店的故事/老板反差/行业秘密 |
| 让人觉得我专业 | 干货类：怎么选店/怎么避坑/成本揭秘 |
| 直接拉人到店 | 促销类：限时福利/新品体验/到店送 |
| 随便，你帮我想 | 根据诊断结果推荐 |

**与诊断结果联动：** 如果报告说「店能接人，问题在拉人」→ 选促销类。如果「店接不住人」→ 先改店，再拍视频。

#### Step 2：选Hook公式（开头5种）

| 公式 | 适合场景 | 示例（餐饮版） |
|------|---------|-------------|
| 🏷️ **标签型** — 地域+品类锁定周边 | 日常更新，快速获客 | 「很多杭州的朋友刷到我，刻意来我家吃火锅」 |
| 🔢 **数字型** — 具体数字制造信息差 | 干货/揭秘 | 「月流水30万，到手不到3万，钱去哪了？」 |
| ⚡ **冲突型** — 认知反差 | 出圈爆款 | 「千万别买XX，除非你看完这条」 |
| 🎁 **利益型** — 直接给好处 | 促销活动 | 「花9.9吃了3个菜，你敢信？」 |
| 💗 **共鸣型** — 戳痛点 | 建立信任 | 「每次请客都找不到合适的店？我也是」 |

**如何选：** 促销活动 → 利益型／标签型。出圈涨粉 → 冲突型／数字型。日常维护 → 标签型／共鸣型。

#### Step 3：用Vikki 5条铁律写正文

| # | 铁律 | 错误写法 → 正确写法 |
|---|------|-------------------|

| 4 | **有故事** | ❌「食材新鲜」→ ✅「客人说这鱼就像刚从河里蹦上来的」 |
| 5 | **两个圆重叠** | ❌「全场5折」→ ✅「到店才有、限今天、每人限1份」 |

| 完播率 | XX | XX | 开头/结尾/时长？ |

**关键问题：**
- 实际 > 预测 → 低估了什么？下次怎么复制？
- 实际 < 预测 → 高估了什么？哪个环节出问题？
- 连续3条预测不准 → 判断标准需要调

#### 6.2.4 升级标准（可选）

积累5条以上完整（预测+数据+复盘）记录后：
```
升级标准
```
系统根据历史数据自动调整打分权重。

### 6.3 与诊断结果的联动

| 诊断结论 | 优先选题 | 优先Hook | 内容核心 |
|---------|---------|---------|---------|
| 店能接人，问题在拉人 | 促销类 | 利益型/标签型 | 福利+位置+为什么值得来 |
| 店接不住人 | 先改店，不发视频 | — | — |
| 在播但效果差 | 干货类/出圈类 | 数字型/冲突型 | 信息差+反认知 |
| 没时间打理 | 手机随手拍 | 标签型 | 真实日常，不求完播率 |

## ⚠️ 内容创作常见错误

| 不要做 | 为什么 |
|--------|--------|
| 一次写太长（超过90秒） | 本地生活视频30-60秒完播率最高 |
| 开头不说人话 | 前3秒抓不住人，后面白写 |
| 只写产品不写场景 | 顾客在乎产品能帮他解决什么，不是产品本身 |
| 发完就不管了 | 不复盘，下一期跟上一期没区别 |
| 发之前改预测 | 预测不准没事，改预测才是问题 |
