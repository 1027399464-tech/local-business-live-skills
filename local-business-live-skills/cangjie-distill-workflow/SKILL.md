# 内容蒸馏工作流 (Content Distill Workflow)

把一本书、一段视频逐字稿、一场直播复盘、一篇长文里的方法论，蒸馏成可执行的 Hermes Skill。

本工作流受 [cangjie-skill](https://github.com/kangarooking/cangjie-skill) 的 RIA-TV++ 方法论启发，针对直播运营/本地生活商家场景做了定制适配。

## 触发条件

用户说这些时触发：
- "帮我把这个X内容蒸馏成skill"
- "把XX整理成方法论"
- "这段内容能做成skill吗"
- "把我直播复盘的方法论抽出来"

## 工作流

### Step 1: 确认是否已有现成

**先查 GitHub 上有没有现成的 skill 包：**
- https://github.com/kangarooking/ （爆款文案/1000个铁粉/疯传等）
- https://github.com/qbdx-hub/（孙子兵法/明朝那些事儿等）
- 如果有现成的 → **推荐直接用**，不要重复造

### Step 2: 如果没有现成的，自己蒸馏

按 RIA-TV++ 六步法处理源内容：

**第一步：整体理解**
- 源内容的结构是什么？
- 核心论点/方法论是什么？
- 它回答了什么"好问题"？
- 产出：一份简短的结构概述

**第二步：提取方法论单元**
从源内容中提取5类素材：
- **框架** — 有步骤/阶段/层次的方法
- **原则** — 可迁移的判断准则
- **案例** — 能说明方法论的具体例子
- **反例** — 什么情况下会失效
- **术语** — 核心概念定义

**第三步：三重验证**
每个提取的方法论单元必须通过3问：
1. **跨域佐证** — 源内容中至少有2处印证这个方法论
2. **预测力** — 它能回答内容里没明说但合理的新问题吗？
3. **独特性** — 这不是"常识"，是有独特洞见的东西

**第四步：RIA++E+B 结构化**
每个通过验证的单元写成：
- **R（原文）** — 源内容的原始引用
- **I（方法论）** — 用自己的话把核心方法写清楚
- **A1（案例）** — 源内容里的实际应用案例
- **A2（触发场景）** — 用户在什么场景下会需要这个skill
- **E（可执行步骤）** — 具体操作步骤（3-5步）
- **B（边界）** — 什么时候不能用+失败模式

**第五步：建立关系**
- 哪些skill依赖其他skill？
- 哪些skill互相对比/补充？
- 产出：依赖图 + 推荐学习顺序

**第六步：测试**
- 设计至少3个测试prompt
- 检查每个skill在测试prompt下能否给出正确的方法指导

### Step 3: 写成Hermes Skill格式

每个skill的SKILL.md格式：
```yaml
---
name: <skill-name>
description: |  一句话描述
source: <来源>
tags: [标签1, 标签2]
related_skills: [关联的其他skill]
---

# 标题

## R — 原文 (Reading)
...
## I — 方法论骨架 (Interpretation)
...
## A1 — 应用案例 (Application)
...
## A2 — 触发场景 (Trigger)
...
## E — 可执行步骤 (Execution)
...
## B — 边界 (Boundary)
...
```

### Step 4: 注册到Hermes

用 skill_manage(action='create') 创建为正式skill。

## 注意事项

- 不用每个候选都变成skill——通不过三重验证的丢掉
- 方法论要有"可执行性"才有价值，纯观点/感想不配做skill
- 边界（B）是最值钱的部分——大多数笔记只记了"怎么做"，没记"什么时候不能做"
- 先做小范围测试（2-3个skill），验证效果后再大规模蒸馏
- 蒸馏自己的内容时注意隐私
