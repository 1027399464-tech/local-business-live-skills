# 直播复盘六部曲

## R — 原文 (Reading)

> 核心目标：通过系统化复盘，精准定位问题，优化直播全流程，实现稳定增长
> 
> — 李鑫直播知识库

直播复盘六部曲从结果到过程，从内部到外部，构成完整诊断闭环：

```
数据复盘 → 节奏复盘 → 商品复盘 → 人群复盘 → 短视频复盘 → 流量结构复盘
   ↓          ↓          ↓          ↓          ↓            ↓
定基调     定成败     找爆款     判精准     挖流量       调比例
```

---

## I — 方法论骨架 (Interpretation)

直播复盘六部曲是一套**由结果到过程、由内部到外部的递进诊断系统**。每一部都依赖前一步的输出作为输入：

**第一步 数据复盘（定基调）：** 先看大盘，确认直播间是否在及格线上。同时看电商数据（转化率/GPM/连单率/老粉占比）和人氣数据（转粉率/停留时长），识别最明显的短板。

**第二步 节奏复盘（定成败）：** 成交节奏决定推流节奏。核心判断：开播极速流接住了没有？是否出现了"一波流"？福利品放单量是否超过在线5%？

**第三步 商品复盘（找爆款）：** 聚焦两个核心指标——商品曝光点击率（15%合格，25%+爆款）和点击成交转化率（高客单>5%，中低10-15%）。诊断方向：曝光高点击低→中控/话术问题；点击高转化低→价格/信任问题。

**第四步 人群复盘（判精准）：** 对比看播用户和购买用户的人群画像是否一致。八大人群均勻无主力=标签混乱，需微付费修正。

**第五步 短视频复盘（挖流量）：** 筛选出基础播放>5000、点击率>5%、5秒完播>30%的视频，开播前抖加加热，开播后千川放大。

**第六步 流量结构复盘（调比例）：** 对比各渠道流量占比和支付金额占比，判断人群精准度。微付费直播间自然流占比50-70%为健康。

**六步的递进逻辑：**
- Step 1-2 → 诊断"直播间本身"的问题（数据基线和节奏）
- Step 3-4 → 诊断"货和人"的问题（商品效率和人群精度）
- Step 5-6 → 诊断"流量来源"的问题（短视频引流和整体结构）

## A1 — 应用案例 (Application)

### 案例 1：微付费直播间数据复盘
**场景：** 某本地生活商家直播间，场均GPM只有客单价的5倍，自然流占比持续下降
**分析步骤：**
1. 数据复盘 → GPM不达标（客单价5倍 < 微付费标准的10倍），自然流难获推流
2. 节奏复盘 → 开播极速流承接不好，每轮放单量超过在线10%，人群跑偏
3. 商品复盘 → 商品曝光点击率8%（合格线15%），中控弹窗不及时
4. 人群复盘 → 看播和购买人群画像不一致，标签混乱
5. 短视频复盘 → 引流视频5秒完播率仅15%，远低于30%合格线
6. 流量结构复盘 → 付费占比已达38%（微付費<30%限制），压制自然流
**结论：** 核心问题顺序：短视频引流质量 → 中控弹窗节奏 → 放单量过大跑偏人群 → GPM不达标锁死推流

### 案例 2：老粉成交占比过高诊断
**场景：** 自然流直播间老粉成交占比40%（正常<20%）
**分析：**
1. 从数据复盘发现问题（老粉占比异常）
2. 从人群复盘找到原因（无主力人群，标签混乱）
3. 解决方案：切换至第四步→用微付费修正标签→第二步调整节奏→等待自然流恢复
**迭代点：** 不是从头开始六步，而是从匹配的步骤切入

## A2 — 触发场景 (Trigger)

### 你会在什么场景下需要这个 skill?

1. **播后复盘** — 刚下播，需要快速定位这场哪里出了问题
2. **商家诊断** — 帮客户看数据时，需要系统化分析而不是凭感觉
3. **连续几场数据下滑** — 不是单场问题，需要从整体结构找根因
4. **自然流被压制** — 付费占比升高、自然流减少，需要流量结构复盘

### 语言信号
- "我这场数据不太好，帮我分析一下"
- "最近几场直播感觉不对劲"
- "帮我看看这个商家的直播数据"
- "这个直播间自然流越来越少了"
- "老粉占比太高怎么办"

### 与相邻 skill 的区分
- 与 `千川标准计划搭建` 的关系：本 skill 诊断直播间问题后，如果需要通过投放解决，调用千川计划搭建 skill
- 与 `五维四率` 的关系：五维四率是本 skill 第一步（数据复盘）的核心工具，两者互补

## E — 可执行步骤 (Execution)

### 完整复盘流程（约30分钟）

**第1步：数据复盘（5分钟）**
- [ ] 记录转化率（低客单20%/高客单10%为合格）
- [ ] 计算GPM/客单价倍数（微付费10-15倍/重付费30-50倍）
- [ ] 查看连单率（成交件数/人数，越高越好）
- [ ] 记录老粉成交占比（正常<20%/高客单可放宽至50%）
- [ ] 记录转粉率（3%+合格）
- [ ] 记录平均停留时长（单身35秒+/多品50秒+）
- [ ] 标出不合格指标 → 进入第2步

**第2步：节奏复盘（5分钟）**
- [ ] 回放开播前15分钟：极速流来了没有？接了还是掉了？
- [ ] 检查放单量：是否超过在线人数5%？
- [ ] 检查话术框架：是否有连续2-3轮成交数据小递增？
- [ ] 识别是否出现"一波流" → 进入第3步

**第3步：商品复盘（5分钟）**
- [ ] 找出商品曝光点击率最高的SKU（15%合格/25%+爆款）
- [ ] 找出点击成交转化率最高的SKU（高客单>5%/中低10-15%）
- [ ] 诊断异常：曝光高点击低→中控弹窗问题；点击高转化低→价格/信任问题
- [ ] 确定主推款 → 进入第4步

**第4步：人群复盘（5分钟）**
- [ ] 对比看播用户 vs 购买用户的人群画像
- [ ] 确认八大人群分布：是否有一支主力人群？
- [ ] 如果人群混乱 → 计划用微付费修正标签
- [ ] 如果标签精准 → 进入第5步

**第5步：短视频复盘（5分钟）**
- [ ] 筛选基础播放>5000的视频
- [ ] 确认点击率>5%、5秒完播>30%
- [ ] 优质视频→开播前1小时抖加加热，开播后千川放大
- [ ] 无优质视频→下一场开播前解决素材问题

**第6步：流量结构复盘（5分钟）**
- [ ] 对比各渠道流量占比 vs 支付金额占比
- [ ] 判断付费占比是否超标（微付费<30%）
- [ ] 超标→需降低付费比例或提升自然流承接
- [ ] 完成六步→生成优化行动清单

## B — 边界 (Boundary)

### 不要在这个场景下使用

1. **新号前3场** — 数据样本太小，六部曲的合格线对冷启动直播间不适用。新号先用"冷启动检查清单"。
2. **纯付费直播间** — 流量结构复盘的健康比例完全不同，自然流占比参考值不适用。
3. **娱乐/才艺直播间** — 指标体系与带货直播间完全不同（不考核交易力），本 skill 仅适用于抖音电商带货。
4. **数据工具不可用时** — 没有电商罗盘/千川后台数据时，本 skill 的指标体系无法执行。

### 常见失败模式

- **跳过前两步直接看商品** — 人气数据不合格时，商品复盘的数据根本没有统计意义（停留太短无法测品）
- **只看单场不做趋势** — 一场的数据噪音很大，至少拉3-5场对比才有判断价值
- **把合格线当目标** — 合格线是"通过门槛"，不是"优秀标准"。追求增长需要逐层提高标准

### 作者盲点

- 该体系基于2025年抖音算法（赛马机制+实时推流），2026年已转向"7天慢推流+首日不考核GMV"，第一步的GPM权重可能需要下调
- 视频号/快手等平台的复盘体系不同（指标体系不同），不可直接套用
